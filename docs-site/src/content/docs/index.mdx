## The problem

Your team has Azure AI Foundry deployed. Developers need API keys to hit it through APIM. But there's no built-in way to:

- Give each developer or group a **monthly token budget** that actually enforces
- **Provision and revoke** APIM subscription keys without portal access
- Let developers **self-serve quota increases** with an approval workflow
- See at a glance who is burning tokens and who is hard-stopped

FoundryGate is the missing operations layer.

## How it works

<CardGrid>
  <Card title="Per-developer quotas" icon="bars">
    Five-level quota resolution: individual override → group policy → system default. When a developer hits their limit, their APIM subscription is **suspended** — not just flagged.
  </Card>
  <Card title="APIM key lifecycle" icon="key">
    Admins provision, rotate, and revoke Azure API Management subscription keys without touching the portal. Developers can rotate their own key from the self-service dashboard.
  </Card>
  <Card title="Monthly hard reset" icon="list-format">
    An Azure Function fires at 00:01 UTC on the 1st. Every active developer gets a fresh allocation. Suspended subscriptions are automatically re-enabled.
  </Card>
  <Card title="Foundry model management" icon="setting">
    Admins can deploy and remove Azure AI Foundry model deployments from the FoundryGate UI. New deployments are immediately reachable via existing keys — no APIM changes needed.
  </Card>
</CardGrid>

## Stack at a glance

| Layer | Technology |
|---|---|
| API | ASP.NET Core 10 · Azure Container Apps |
| Background jobs | Azure Functions (.NET 10 isolated, Flex Consumption) |
| Database | Azure SQL · EF Core 10 · dacpac deployment |
| Frontend | Blazor WebAssembly · MudBlazor |
| Auth | Microsoft Entra ID · `Microsoft.Identity.Web` |
| APIM integration | `Azure.ResourceManager.ApiManagement` |
| Foundry provisioning | `Azure.ResourceManager.CognitiveServices` |
| Docs | Astro · Starlight · GitHub Pages |
